# Регрессионный чек-лист

> Полная проверка всех фич перед релизом. Выполняется после любых изменений в коде.

## 1. Загрузка файлов

- [ ] Drag-and-drop MP3 — добавляется в список
- [ ] Drag-and-drop MP4 — добавляется в список
- [ ] Drag-and-drop неподдерживаемого формата (.txt) — игнорируется
- [ ] Выбрать файлы через диалог — 1, 5, 50 файлов
- [ ] Дубликаты (одинаковое имя+размер) — не добавляются
- [ ] Удалить файл из списка (✕) — счётчик уменьшается
- [ ] Список скрыт, когда файлов нет
- [ ] Счётчик: «Добавлено файлов: N»
- [ ] Drop вне drop-zone — файл НЕ открывается в браузере
- [ ] Unicode в имени файла — корректно
- [ ] Пробелы и спецсимволы в имени файла — корректно

## 2. Контекст

- [ ] Пустой контекст — обработка проходит без ошибок
- [ ] Контекст с текстом — влияет на транскрипцию
- [ ] Длинный контекст (500+ символов) — не ломает pipeline
- [ ] <script> в контексте — XSS защита (не выполняется)

## 3. Формат вывода

- [ ] DOCX по умолчанию
- [ ] Переключение DOCX ↔ TXT
- [ ] DOCX создаётся, открывается без ошибок
- [ ] TXT создаётся, UTF-8 без кракозябр
- [ ] Имя файла = `{оригинал}.{format}`
- [ ] Кнопка «Открыть папку» работает

## 4. UI

- [ ] Тёмная тема по умолчанию, переключение на светлую
- [ ] Модалка: открыть (?) → ✕ / Escape / клик вне → закрыть
- [ ] Спинсер: скрыт → обработка → показан → завершение → скрыт
- [ ] Лог: появляется при старте, автоскролл, очистка при новом запуске
- [ ] Кнопка «Обработать»: disabled без файлов, enabled с файлами, скрыта во время обработки
- [ ] Кнопка «Отмена»: скрыта до обработки, видна во время, скрыта после

## 5. Pipeline

- [ ] 1 MP3 → detect → transcribe → enhance → save → cleanup
- [ ] 1 MP4 → detect → transcribe → enhance → save → cleanup
- [ ] 3 файла → все проходят последовательно, прогресс корректен
- [ ] Тихая запись — компрессия (compand) применяется
- [ ] 3-pass enhancement pass 1 → 2 → 3
- [ ] Chunking (>3000 символов) — текст разбивается
- [ ] Safety ratio — результат не < 60% и не > 140%

## 6. Отмена

- [ ] Отмена на каждом шаге pipeline (detect / transcribe / enhance / save / cleanup)
- [ ] Частичные результаты сохранены
- [ ] UI восстанавливается после отмены
- [ ] Двойная отмена — без ошибки

## 7. Видео

- [ ] MP4, AVI, MKV, MOV, WEBM — аудио извлекается
- [ ] Видео без аудиодорожки — понятная ошибка
- [ ] Битое видео — fallback с логом
- [ ] Оригинальный видеофайл удаляется после обработки

## 8. GPU / Статус-бар

- [ ] Статус-бар отображается при загрузке
- [ ] Правильная комбинация: GPU × CUDA × Ollama × Qwen
- [ ] /api/gpu возвращает JSON
- [ ] /api/ollama возвращает JSON

## 9. SSE

- [ ] SSE стрим открывается после /api/files
- [ ] log / progress / done / error / cancelled — все события обрабатываются
- [ ] keepalive каждые 30 сек при пустой очереди
- [ ] Обрыв соединения — onerror

## 10. API

- [ ] POST /api/files — 1 и N файлов, валидация
- [ ] GET /api/status/{id} — SSE
- [ ] POST /api/cancel/{id} — cancel
- [ ] GET /api/download/{id}/{file} — скачивание
- [ ] GET /api/gpu — JSON
- [ ] GET /api/ollama — JSON
- [ ] GET /api/open-output — Explorer

## 11. Установка / Сборка

- [ ] build_zips cpu — собирается без ошибок
- [ ] build_zips cu126 — собирается без ошибок
- [ ] Bootstrap — idempotent
- [ ] Launcher — запуск без консоли, убийство python.exe при закрытии
- [ ] Uninstall — все компоненты удаляются
