# Смоук-чек-лист

> Минимальная проверка критического пути перед каждым релизом.
> Время выполнения: ~5-10 минут.

## 🛠️ Чем проверять

Открой приложение (через run.bat или ярлык), если не работает — открой http://127.0.0.1:{порт} в браузере.

Просто пройди по пунктам. Берёшь один mp3 с речью (любой, хоть диктофонную запись), тащишь в окно, жмёшь Обработать, ждёшь, открываешь output.

Если на каком-то шаге что-то пошло не так — открой F12 (через браузер), Console смотри ошибки, Network — SSE-события.

Смоук должен проходиться за 5 минут. Если дольше — что-то не так.

## Критический путь: загрузить → обработать → получить результат

- [ ] Открыть приложение — интерфейс загружается без ошибок
- [ ] Список файлов пуст, кнопка «Обработать» disabled
- [ ] Перетащить 1 MP3 файл (любой, с речью) — появляется в списке
- [ ] Счётчик: «Добавлено файлов: 1»
- [ ] Нажать «Обработать» — появляется спинсер, лог, кнопка «Отмена»
- [ ] В логе: шаги Detect → Transcribe → Enhance → Save
- [ ] После завершения: спинсер скрыт, кнопки восстановлены
- [ ] В папке output/ появился .docx файл
- [ ] .docx открывается, содержит распознанный текст
- [ ] Нажать «📁 Открыть папку» — открывается Explorer

## Дополнительно (если есть GPU)

- [ ] Статус-бар: GPU NVIDIA + Torch CUDA — зелёный цвет
- [ ] Или: GPU не обнаружена + Torch CPU — нейтральный цвет

## Edge cases (если время позволяет)

- [ ] 1 MP4 видео — обрабатывается, результат в output/
- [ ] Нажать «Отмена» во время обработки — обработка прерывается
- [ ] Тёмная → светлая тема — все элементы перекрашиваются
- [ ] Модалка «?» открывается и закрывается
