# Чек-лист: Формат вывода

## Техники: EP, State Transition

### Выбор формата

- **EP:** DOCX (default), TXT
- [ ] Формат по умолчанию — DOCX (выбран при открытии)
- [ ] Переключить на TXT — формат меняется
- [ ] Переключить обратно на DOCX — работает

### Сохранение DOCX

- [ ] Обработать файл с форматом DOCX — файл создаётся в output/
- [ ] DOCX открывается в Word / LibreOffice
- [ ] Имя файла: `{оригинальное_имя}.docx`
- [ ] Текст разбит на абзацы
- [ ] Кириллица отображается корректно
- [ ] Спецсимволы, кавычки, тире — корректно

### Сохранение TXT

- [ ] Обработать файл с форматом TXT — файл создаётся в output/
- [ ] TXT открывается в блокноте, кодировка UTF-8
- [ ] Имя файла: `{оригинальное_имя}.txt`
- [ ] Кириллица без кракозябр

### Папка output

- [ ] Папка `output/` создаётся при первом запуске, если её нет
- [ ] Нажать «📁 Открыть папку» — открывается Explorer с output/
- [ ] После 3 обработок в папке 3 файла (1 на каждый входной файл)
- [ ] Длинные имена файлов — сохраняются без обрезки (проверить BVA: > 255 символов для полного пути)

### State Transition: множественная обработка

- [ ] 3 файла, формат DOCX → создаётся 3 .docx
- [ ] 3 файла, формат TXT → создаётся 3 .txt
- [ ] Смешанные форматы не поддерживаются — один формат на весь батч
