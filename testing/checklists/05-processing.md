# Чек-лист: Pipeline обработки

## Техники: State Transition, EP, BVA, таблица решений

### State Transition — 5 шагов pipeline

Для каждого файла проходятся шаги:
1. **DetectPreprocess** → 2. **Transcribe** → 3. **Enhance** → 4. **Save** → 5. **Cleanup**

- [ ] Файл проходит все 5 шагов последовательно
- [ ] Лог отображает прогресс по каждому шагу
- [ ] После завершения pipeline — спиннер скрыт, кнопки восстановлены

### Detect & Preprocess

- **EP (громкость):** нормальная (> -20dB) vs тихая (< -20dB)
  - [ ] Нормальная запись — только highpass/lowpass/loudnorm
  - [ ] Тихая запись — дополнительно compand (компрессия)
- [ ] ffmpeg применяет: highpass 80Hz, lowpass 8kHz, loudnorm, 16kHz, mono
- [ ] Если ffmpeg падает — fallback: используется оригинальный файл, в логе предупреждение
- **Негатив:** битый аудиофайл — fallback с логом
- **Cancel:** ffmpeg процесс убивается, pipeline прерывается

### Transcribe (Whisper)

- [ ] Модель: medium, язык: ru
- [ ] VAD фильтр включён (пропускает тишину)
- [ ] Beam search: 5
- [ ] Передаётся контекст (initial_prompt)
- [ ] Лог: количество символов после транскрибации
- **Негатив:** аудио без речи — пустой результат с логом? (проверить)
- **Негатив:** файл с музыкой без слов — что возвращает Whisper?
- **Cancel:** прерывается между сегментами

### Enhance (Qwen 2.5)

- **3-pass multi-pass:**
  - [ ] Pass 1: очистка (орфография, пунктуация, повторы)
  - [ ] Pass 2: стиль (грамматика, порядок слов)
  - [ ] Pass 3: структура (абзацы, диалоги)
- [ ] Топик определяется перед enhancement (история, образование, техника, повседневный)
- [ ] Диалоги форматируются: тире → [СПИКЕР N]:
- **Chunking:** текст > 3000 символов — разбивается на чанки
  - [ ] Каждый чанк проходит 3 passes
- **Safety ratio:** результат < 60% или > 140% оригинала — rejected, сохраняется предыдущий текст
  - [ ] Проверить с intentionally short/long результатом от Qwen
- **Негатив:** Ollama недоступна — ошибка с fallback?
- **Негатив:** Ollama таймаут — повторная попытка
- **Cancel:** прерывается между чанками/proходами

### Save

- [ ] Пустой результат — файл НЕ создаётся, лог: "Нет текста для сохранения"
- [ ] DOCX — создаётся через python-docx
- [ ] TXT — сохраняется как UTF-8

### Cleanup

- [ ] Временные WAV-файлы удаляются после обработки
- [ ] Временные файлы удаляются даже при ошибке на поздних шагах

### Множественные файлы

- **BVA:** 1 файл, 2 файла, 10 файлов
- [ ] Файлы обрабатываются последовательно (один за другим)
- [ ] Progress event: {current}/{total} — корректно обновляется
- [ ] ETA рассчитывается и отображается
- [ ] После всех файлов — DoneEvent

### Негативные сценарии

- [ ] Файл > 1 ГБ — рост времени, но не падение
- [ ] Битый WAV — fallback/ошибка с логом
- [ ] Нет места на диске — ошибка сохранения
- [ ] Ошибка на одном файле — pipeline продолжается со следующим
