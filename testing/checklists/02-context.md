# Чек-лист: Контекст (initial_prompt)

## 🛠️ Чем проверять

**DevTools (через браузер на http://127.0.0.1:{порт}, F12):**
- Elements — найди textarea, смотри атрибут value, меняется ли при вводе
- Network — найди POST /api/files, в Form Data поле initial_prompt — туда должен улететь текст

**curl (если есть Git Bash или просто cmd):**
Отправить контекст со спецсимволами напрямую в API:
curl -X POST -F "files=@test.mp3" -F "initial_prompt=<script>alert(1)</script>" http://127.0.0.1:5000/api/files
Если curl нет — поставь Git Bash или используй Postman.

**Спецсимволы:**
- <script>alert(1)</script> — XSS, если выполнится — плохо
- эмодзи — скопируй из браузера или виртуальной клавиатуры Win+.
- unicode — "тест контекста с кавычками «ёлочками» и тире —"

**Notepad++:**
просто чтобы скопипастить unicode-текст и вставить в поле.

## Техники: EP, BVA

### Эквивалентные классы

- **Класс 1:** пустой контекст (нет ввода)
- **Класс 2:** короткий контекст (1 слово, "техника")
- **Класс 3:** средний контекст (1 предложение)
- **Класс 4:** длинный контекст (абзац, 500+ символов)
- **Класс 5:** текст с Unicode/эмодзи

### BVA

- [ ] 0 символов (пусто)
- [ ] 1 символ
- [ ] 1000+ символов (длинный текст)

### Проверки

- [ ] Ввести контекст, обработать файл — контекст влияет на распознавание
- [ ] Поле resizable — тянется по вертикали
- [ ] Контекст не сбрасывается после завершения обработки?
- [ ] Тёмная тема — поле отображается корректно
- [ ] Светлая тема — поле отображается корректно

### Негативные сценарии

- [ ] Ввести спецсимволы (<script>, SQL-инъекции) — не ломают UI и сервер
- [ ] Ввести только пробелы — обрабатывается как пустой
- [ ] Ввести эмодзи и non-BMP символы — не ломают pipeline
