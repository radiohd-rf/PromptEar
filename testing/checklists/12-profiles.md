# Чек-лист: Профили улучшения + Резюме

## 🛠️ Чем проверять

**DevTools (F12 через браузер):**
- Network → POST /api/files → Form Data: поле `profile` — должно совпадать с выбранным
- Network → GET /api/profiles — список профилей с сервера

**Содержимое результата:**
- Распознанный текст глазами — соответствует ли профилю (формальный vs разговорный, полный vs выжимка)
- Для summary: файл `*_summary.md`, 3–5 предложений

**Проводник:**
- output/ — проверь имя файла и формат (.docx/.txt для обычных, .md для резюме)

## Техники: EP, Decision Table

### Выбор профиля (UI)

- [ ] Дропдаун «Профиль» отображается в тулбаре рядом с «Формат»
- [ ] По умолчанию выбран «Разговорный»
- [ ] Доступны: Разговорный, Формальный, Конспект, Резюме (выжимка)
- [ ] Выбор профиля не сбрасывается при загрузке файлов
- [ ] GET /api/profiles возвращает список профилей с сервера

### Decision Table: комбинации «профиль × формат»

| Профиль | Формат DOCX | Формат TXT |
|---------|:-----------:|:----------:|
| Разговорный | [ ] | [ ] |
| Формальный | [ ] | [ ] |
| Конспект | [ ] | [ ] |
| Резюме | [ ] N/A — всегда .md | [ ] N/A — всегда .md |

- [ ] Разговорный + DOCX — файл `{name}.docx`, улучшенный текст
- [ ] Формальный + DOCX — файл `{name}.docx`, формальный/научный стиль
- [ ] Конспект + DOCX — файл `{name}.docx`, лаконичный, со списками
- [ ] Резюме — файл `{name}_summary.md`, выжимка 3–5 предложений

### Профиль «Разговорный» (default)

- [ ] 3 прохода: очистка → стиль → структура
- [ ] В логе отображаются все 3 прохода
- [ ] Текст минимально изменён (исправлена только орфография/пунктуация/структура)
- [ ] Стиль — естественный, разговорный
- [ ] Абзацы разбиты по смыслу

### Профиль «Формальный»

- [ ] 3 прохода: очистка → стиль → структура
- [ ] Текст приведён к формальному/научному стилю
- [ ] Точная терминология, без разговорных оборотов
- [ ] Разделы с заголовками (если тема меняется)
- [ ] Сравнить с «Разговорный» — формальный вариант заметно суше

### Профиль «Конспект»

- [ ] 3 прохода: очистка → стиль → структура
- [ ] Текст лаконичный, без «воды»
- [ ] Есть маркированные списки и тезисы
- [ ] Группировка близких по смыслу утверждений
- [ ] Сравнить с исходником — конспект короче полного текста

### Профиль «Резюме (выжимка)»

- [ ] 2 прохода: очистка → summary (стиль и структура пропущены)
- [ ] В логе: «Проход 1/2: Очистка», «Проход 2/2: Резюме»
- [ ] Результат — 3–5 предложений
- [ ] Файл: `{name}_summary.md`, формат Markdown
- [ ] Файл НЕ создаётся как .docx или .txt
- [ ] Содержит суть текста: о чём речь, кто говорит, ключевые факты
- [ ] Имена, даты и числа сохранены
- [ ] Нет интерпретаций/выводов от модели

### Backend API

- [ ] GET /api/profiles — `{"current":"default","profiles":[...]}`
- [ ] POST /api/files с `profile=summary` — в FormData есть profile
- [ ] POST /api/files с `profile=summary` — в задаче profile=summary
- [ ] POST /api/files без profile — profile=default (обратная совместимость)

### Регрессия (ничего не сломалось)

- [ ] Старый формат POST /api/files без `profile` — обрабатывается как default
- [ ] Обработка с профилем по умолчанию — поведение идентично v0.11.2
- [ ] Все форматы (DOCX/TXT) продолжают работать
- [ ] Тёмная/светлая тема — дропдаун профилей виден и читаем
- [ ] Горячие клавиши — дропдаун не перехватывает фокус
- [ ] SSE-события — для summary не ломается EventSource
